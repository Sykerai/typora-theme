# Phycat Forest 正文响应式宽度设计

## 问题

`phycat-forest` 当前只为 `#write` 设置了 `max-width: 1200px`。当 Typora 从大屏移动到较小屏幕时，正文容器会缩小到可用区域的 100%，网格背景因此紧贴编辑区边缘。

## 目标

- 小屏下正文容器占编辑区可用宽度的 90%，左右各保留约 5%。
- 大屏下正文最大宽度仍为 1200px。
- 正文始终水平居中。
- HTML 导出采用相同的“90% 宽度、最大 1200px”逻辑。
- 打印规则以及现有表格、代码字体、提示块和工具栏样式不变。

## 设计

保留 `--max-width: 1200px`，在主题入口文件中覆盖正文容器：

```css
#write {
  width: 90%;
  max-width: var(--max-width);
  margin-left: auto;
  margin-right: auto;
}
```

HTML 导出继续使用 `width: 90%` 和相同的最大宽度变量，并覆盖共享主题在小屏导出时设置的 `width: 100%`。打印媒体规则带有独立的 `!important` 声明，不作修改。

## 验收

- 可用宽度小于约 1333px 时，正文宽度等于可用宽度的约 90%。
- 可用宽度大于约 1333px 时，正文宽度不超过 1200px。
- 编辑视图、普通 HTML 导出和打印规则互不冲突。
- 原有主题调整仍然存在，CSS 大括号保持平衡。
